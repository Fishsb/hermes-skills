# Knowledge Base Compression

> 以「不丢失任务执行效果」为唯一标准压缩知识库结构。

## 触发方式

当用户说出以下触发词时加载此 skill：
- **上下文压缩**
- **记忆压缩**

## 执行流程

### 0️⃣ 启动前确认

**必须先询问用户是否要执行压缩操作**。不可自动执行。

```
[压缩分析完成]
发现可清理项：N 条
预计 Memory 从 X% → Y%
预计 User Profile 从 X% → Y%
是否执行压缩？(Y/n)
```

用户确认后才继续。

### 1️⃣ 读取当前状态

```python
# 在 skill 中通过 memory/terminal 工具获取当前各存储系统的容量和条目
```

需要检查的对象：

| 对象 | 检查工具 | 判定标准 |
|------|---------|---------|
| Memory | 阅读当前 system prompt 中的 MEMORY 块 | 条目数 + 已用字符 |
| User Profile | 阅读当前 system prompt 中的 USER PROFILE 块 | 条目数 + 已用字符 |
| SOUL.md | `read_file ~/AppData/Local/hermes/SOUL.md` | 是否有与 PLUR/Skill 重复的规则 |
| Skills 目录 | `ls ~/AppData/Local/hermes/skills/` | 空类别、平台不匹配、类别无 DESCRIPTION.md |
| PLUR | `mcp_plur_plur_status`, `mcp_plur_plur_doctor` | 嵌入状态、重复 engram |

### 2️⃣ 逐项判定（删除标准）

**核心原则：只删对任务执行没有帮助的数据。**

#### Memory 删除标准
- ❌ 会话完成态（"PR #XXXX 已提交"、"修复了 bug Y"）
- ❌ 一次性 cleanup 记录
- ❌ 与 SOUL.md 或 PLUR engram 完全重复的行为规则
- ❌ Skill 内已有完整文档的参数细节（如 electron-builder 步骤）
- ✅ 保留：知识库指针、环境稳态事实、网络替代方案、工具配置参数

#### User Profile 删除标准
- ❌ 与 SOUL.md/PLUR 完全重复的行为规则
- ❌ 已删除 profile 的历史记录
- ❌ UI 偏好细节、预算信息
- ❌ 应在 skill 中定义的过程性偏好
- ✅ 保留：核心身份、交互风格偏好、安全规则、方法论、服务器信息、用户决策记录

#### SOUL.md 删除标准
- ❌ 与 PLUR engram 完全重复的行为规则
- ✅ SOUL.md 应只存放：人格定义、交流风格、语气偏好

#### Skills 删除标准
- ❌ 平台不匹配（Windows 上的 apple/ 类 skill）
- ❌ 空类别（有 DESCRIPTION.md 但无实际 skill）
- 🔀 内容高度重复的 skill

### 3️⃣ 执行

**使用 `memory()` 工具**（不是 write_file）：

```python
# Memory 操作
memory(action='remove', target='memory', old_text='...唯一标识...')

# User Profile 操作  
memory(action='remove', target='user', old_text='...唯一标识...')
memory(action='replace', target='user', old_text='...', content='...压缩后的内容...')
```

注意：
- 每次 `memory()` 调用只能操作一条 entry
- 需要多次调用才能完成全部清理
- 用 `old_text` 参数匹配唯一标识，勿在一条 old_text 中包含多个条目的内容
- SOUL.md 用 `write_file` 整体重写（覆盖）
- Skills 目录用 `terminal` 执行 `rm -rf`
- Skill 合并用 `skill_manage(action='patch')` + `skill_manage(action='delete', absorbed_into='...')`

### 4️⃣ 验证

执行完毕后重新读取各系统状态，确认：
- Memory / User Profile 容量显著下降
- SOUL.md 内容正确
- Skills 目录结构合理
- 关键任务信息未被误删

### 5️⃣ 报告

向用户报告压缩结果，格式：

```
Memory: X% → Y% (删除 N 条)
User Profile: X% → Y% (删除 N 条, 压缩 N 条)
SOUL.md: 已优化
Skills: 合并 N 组, 删除 N 空/废弃类
PLUR: 嵌入状态 X
```

## Pitfalls

- **memory() 有字符上限**（2,200 chars）。替换内容过大会被拒绝。解决办法：`remove` + `add` 两步走。
- **SOUL.md 可能被子 Agent 同时修改**。改写前先 `read_file` 确认当前内容，覆盖写入后验证。
- **`rmdir` 无法删除有文件的空目录**（如只有 DESCRIPTION.md 的空类）。用 `rm -rf`。
- **skill_manage() 对根目录的技能**（如 agent-reach）无法写入更新，需用 terminal 直接操作文件。
- **APPLE 类 skill** 仅 Windows 上可安全删除；macOS 上应保留。
- **不要在一个 old_text 中跨多个 entry** — memory() 的 old_text 匹配的是单一 entry 的头几个字符。
