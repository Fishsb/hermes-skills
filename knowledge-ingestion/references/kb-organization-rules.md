# 知识库组织规范 (Knowledge Base Organization Rules)

> 归档知识库环节的参考文件，用于确保 Wiki 笔记写入后保持结构整洁、链接完整、索引健壮。
> 加载方式: `skill_view(name='skill-optimization', file_path='references/kb-organization-rules.md')`

## 一、文件夹职责边界

每个 `wiki/` 子目录有明确的内容边界，归档时按此判定目标目录：

| 目录 | 应放内容 | 不应放内容 |
|------|---------|-----------|
| **工具/** | Hermes/Codex/Claude/插件等具体工具的配置、排障、使用指南 | 工作流方法、环境配置、项目调研、会话归档 |
| **工作流/** | 自动化流程、工程方法论、文档处理流水线、审计工作流 | 具体工具排障、环境安装、项目调研 |
| **环境/** | WSL/VPS/Windows 系统配置、安装部署、环境迁移、Fcitx5 等系统级配置 | 具体工具的使用教程、技能备份 |
| **配置/** | Agent 规则、Profile 配置、记忆策略、知识库规范、索引、备份、历史记录 | 工具层面的配置细节（应归 工具/） |
| **项目/** | 技术调研、GitHub 项目分析、研究报告、竞品分析、A/B 方案选型文档 | 工具文档本身、已归档的会话原始提取 |
| **skill/** | Skill 备份（完整 SKILL.md）、开发记录、索引 | 技能相关的分析文档应放 skill/Skill开发/ |
| **概念/** | 通用概念笔记、跨领域知识点 | 操作指南、教程类内容 |
| **医学/** | 医药健康知识 | 非医学内容 |
| **记忆/** | 归档提炼后的原始记忆源、会话归档目录 | 正式知识笔记（应放对应目录） |
| **记忆/会话归档/** | 所有 auto-extracted 会话提取文件、历史归档 | 正式知识笔记 |

## 二、归档文件禁区

**auto-extracted 文件禁止写入内容目录。** `smart-archive.py` 提取的 `tags: ["分类", "archived", "auto-extracted"]` 文件是原始会话摘录，不是正式知识笔记，必须归入 `记忆/会话归档/`。

### 判定规则

```
文件 tags 含 "auto-extracted"?
  ├─ 是 → 必须写入 记忆/会话归档/，tags 中分类名改为 "会话归档"
  └─ 否 → 按类别映射写入对应内容目录
```

### 例外处理

- `status: superseded` 的旧版规则/设计文档 → 留在原目录，标注 `superseded`，不移动
- `status: closed` 的日志类文件 → 归入 `记忆/`
- 手动编写的 `.md` 笔记即使内容简短，也不属于 auto-extracted

## 三、文件名一致性检查

写入 Wiki 前检查：

```
┌─ 文件名是否反映内容？
│  ✅ "Hermes Desktop 安装排障记录-20260611.md" ← 明确
│  ❌ "测试.md" ← 过泛
│
├─ 文件名是否与文件夹匹配？
│  ✅ "WSL迁移记录.md" → 环境/  (匹配)
│  ❌ "WSL迁移记录.md" → skill/ (不匹配)
│
└─ 文件名是否与现有文件冲突？
   → 同名文件追加 `_archived_<date>.md` 后缀
```

### 常见错位模式

| 文件名关键词 | 正确目录 | 常见错误目录 |
|------------|---------|-------------|
| 含"工作流"、"流程"、"流水线" | `工作流/` | `工具/` |
| 含"环境"、"安装"、"部署"、"迁移"、"WSL" | `环境/` | `配置/`、`工具/` |
| 含"项目"、"调研"、"生态" | `项目/` | `工具/` |
| 含"备份"、"规范"、"规则"、"索引" | `配置/` | `工具/`、`skill/` |
| 含"skill"但为 skill 分析文档 | `skill/Skill开发/` | `工具/` |
| 含"视频总结"、"B站" | 按主题归入对应目录 | `记忆/会话归档/`(仅归档原始提取) |

## 四、链接修复流程

移动文件后必须修复所有指向旧路径的 wiki 链接：

### 步骤

1. **扫描引用** — 在 vault 根目录执行：
   ```bash
   grep -rn "\[\[旧路径/文件名" --include="*.md" .
   ```

2. **主修复目标**: `配置/全库互链索引.md`（全库链接兜底索引）

3. **其他检查目标**:
   - 同文件夹内文件的相对路径引用 `[[../旧目录/文件]]`
   - 使用 `[[文件名]]`（无路径）的链接通常自动存活，需验证文件名唯一性
   - 绝对路径引用 `/mnt/...` 属于硬编码，需手动更新

4. **批量修复命令**:
   ```bash
   sed -i 's|\[\[旧路径/文件|[[新路径/文件|g' "配置/全库互链索引.md"
   ```

5. **验证** — 重新扫描确认零残留：
   ```bash
   grep -rn "\[\[旧路径/文件" --include="*.md" . | grep -v "_temp_session/"
   ```

### 特殊场景

- **维护报告内的路径引用** — 历史审计报告保留原始路径，不修改（非活跃链接）
- **Dataview `FROM` 路径** — 文件移出目录后 Dataview 自动不再显示，无需手动修改
- **`[源于会话: xxx]`** — 纯文字标记，非 wiki 链接，无需修改

## 五、标签修正规则

文件移动后，Frontmatter 中的 `tags` 如果包含旧分类名，需同步更新。

### 需要修正的标签模式

| 旧标签 | 新标签 | 适用场景 |
|-------|-------|---------|
| `["工具", "archived", "auto-extracted"]` | `["会话归档", "archived", "auto-extracted"]` | 归档文件移入会话归档 |
| `["工作流", "archived", "auto-extracted"]` | `["会话归档", "archived", "auto-extracted"]` | 同上 |
| `["环境", "archived", "auto-extracted"]` | `["会话归档", "archived", "auto-extracted"]` | 同上 |
| `["工具", "archived", "superseded"]` | `["会话归档", "archived", "superseded"]` | 已过时文件移入会话归档 |

### 不需要修正的标签

- 不含文件夹名的标签（如 `["AI", "记忆管理", "Agent"]`）
- 多行 YAML 格式的 tags（如 `tags:\n  - type/guide\n  - domain/ai`）— 除非显式包含目录名

## 六、Dataview 索引防护

📂 索引文件的 `FROM` 路径如果包含子目录，应加 `file.folder` 过滤，防止备份文件/子目录内容撑爆索引。

### 安全模式

```sql
-- ❌ 危险: 会递归包含所有子目录文件
FROM "wiki/配置"
WHERE file.name != this.file.name

-- ✅ 安全: 只显示本层文件
FROM "wiki/配置"
WHERE file.folder = "wiki/配置"

-- ✅ 子目录独立区段 (当子目录文件少时可选)
FROM "wiki/配置/维护报告"
```

### 需要过滤的高风险子目录

| 父目录 | 子目录 | 风险 |
|-------|-------|------|
| `配置/` | `Hermes迁移/` | 含 1600+ 备份文件，会撑爆索引 |
| `配置/` | `维护报告/` | 含 7 个审计报告，建议独立区段 |
| `记忆/` | `会话归档/` | 含 30+ 归档文件，建议独立区段 |
| `记忆/` | `_temp_session/` | 暂存 JSON+meta，不应出现在索引中 |

## 七、Frontmatter 校验

写入 Wiki 文件前检查 Frontmatter 格式：

```yaml
# ✅ 正确
---
title: "xxx"
tags: [type1, type2]
---

# ❌ 错误 — 分隔符多了 |
|---
tags: [...]
|---

# ❌ 错误 — 缺少闭合分隔符
---
tags: [...]

(内容...)
```

### 必填字段检查清单

- [ ] Frontmatter 以 `---` 开始和结束（不是 `|---`）
- [ ] `description` 字段存在且不为空
- [ ] `tags` 字段存在，包含至少 1 个分类标签
- [ ] `status` 字段存在（active / archived / superseded）
- [ ] 归档文件含 `source` + `archived` 字段
- [ ] YAML 值中不含未转义的特殊字符（如冒号后缺空格）

## 八、归档后自检清单

每次归档知识库后执行以下检查：

- [ ] 文件写入正确目录（对照文件夹职责边界表）
- [ ] 非 auto-extracted 文件不进 记忆/会话归档/
- [ ] 文件名与内容匹配，不与其他文件冲突
- [ ] 文件移动后全库互链索引已更新
- [ ] 无残留的旧路径 wiki 链接
- [ ] 文件标签与目标目录一致
