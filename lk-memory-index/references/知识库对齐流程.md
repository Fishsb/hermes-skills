# 知识库对齐流程

> 当 `lk-memory-index` skill 中记录的统计数字与实际知识库不一致时的对齐方法。

## 触发条件

- 知识库有明显的内容新增/删除后（新增笔记、会话归档、目录调整）
- 用户说"知识库更新对齐一下"或类似指令
- skill 中文件数标为 `~` 近似值，需要确认

## 对齐步骤

### 1. 获取实际文件数

```bash
cd D:/lk/Obsidian/Hermes
for d in wiki/*/; do
  count=$(find "$d" -name "*.md" | wc -l)
  echo "$d → $count"
done
echo "TOTAL wiki: $(find wiki/ -name '*.md' | wc -l)"
echo "TOTAL all: $(find . -name '*.md' -not -path './.obsidian/*' -not -path './.trash/*' | wc -l)"
```

### 2. 与 skill 中三层结构表对比

skill 文件位置：`C:\Users\lk\AppData\Local\hermes\skills\hermes\lk-memory-index\SKILL.md`

关注三项：
- **三层结构**：`raw/` | `wiki/` | `schema/` 文件数
- **wiki/ 子目录表**：8 个目录各自的文件数
- **总计**：知识库日志中的统计（wiki + raw + schema + _assets/_templates/.workbuddy）

### 3. 找出新增/删除的文件

用 `wiki/log.md` 的最后修改时间筛选新文件：

```bash
find wiki -name "*.md" -newer wiki/log.md -not -path "*/\.*" | sort
```

### 4. 更新 skill

- 更新三层结构表文件数
- 更新 wiki/ 子目录表的文件数
- 如文件有新类型，同时更新"内容"列描述
- 小版本号 +1

### 5. 更新知识库日志

在 `wiki/log.md` 追加变更记录，格式：

```markdown
## YYYY-MM-DD — [变更简述]

**类型**：✏️ 内容修改
**操作人**：AI
**变更内容**：
- 变更条目列表
**文件统计**：详细的分项统计
```

## 注意事项

- **只更新统计数字**，不修改知识库文件本身（除非发现断链或路径错误）
- **Dataview 索引文件无需手动编辑**（自动反映 frontmatter 变化）
- **不要删除历史日志**，只追加新条目
- **版本号规则**：小版本 +1 对应数字修正，大版本 +1 对应架构结构性变化
- 如果发现 `wiki/环境/` 等目录内容描述与实际情况不符（如内容未变但文件名换了），同步更新"内容"列
