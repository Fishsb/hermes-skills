## Hermes 三层记忆架构

Hermes 有两层持久存储 + 一层 skill 展开层：

| 层 | 工具 | 容量 | 内容策略 | 当前占用 |
|---|------|------|---------|---------|
| **memory** | `memory(target='memory')` | 2,200 chars | 技能指针（≤80字/条），不存长细节 | 27 chars (1%) |
| **user profile** | `memory(target='user')` | 1,375 chars | 身份/交互偏好/硬规则/技能指针 | 289 chars (21%) |
| **skill 文件** | `skill_manage` / `skill_view` | 不限 | 长内容、技术细节、工作流程 | 31 skills |

三条约束规则：
1. **不在 memory 放细节** — 只放 ≤80 字 skill 指针，细节进 skill
2. **跨存储区去重** — 同一规则只保留一份（user profile 优先于 memory，skill 优先于 profile）
3. **长内容移入 skill** — 技术细节、配置参数、工作流程 → skill 文件，profile 留指针

## 记忆压缩原则

- MEMORY 只保存 ≤80 字的唤醒指针。
- user profile 只保存影响每次对话判断的核心信息（身份、偏好、硬规则）。
- 细节放 skill 或 Obsidian 知识库。
- 不记录短期任务进度、PR/issue/commit、一次性结果、现实生活信息。
- 修改用户长期规则时，优先更新对应 skill/知识库，再在 MEMORY 中保留指针。
- 跨存储区去重：检查 memory 与 user profile 之间是否有重复条目。重复时保留更合适的那份（user profile 优先于 memory），删除另一份。
